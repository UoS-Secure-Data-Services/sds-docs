# Contribution guidelines

We're trying to establish some guidelines for documentation. This is an evolving list and should be the start of the discussion, not the end.

Current considerations from existing merge requests are as follows:

+ Try to make parameter examples useful but not give away information such as s3 endpoints
+ Make sure that any examples are for the production, not dev, system.
+ If missing out steps will causes errors, describe those errors.
+ Use single quotes for inline code, e.g. \'cd /etc/nginx/\'
+ In Admonitions, use double quotes around title.
